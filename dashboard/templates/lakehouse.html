Fitur Delta Lake

⏱️ Time Travel

Akses versi historis tabel dengan VERSION AS OF atau TIMESTAMP AS OF. Memungkinkan audit kapan data berubah dan perbandingan snapshot berbeda.

spark.read.format("delta") \
  .option("versionAsOf", 0) \
  .load("...feature_store")

🔒 ACID Transactions

Setiap write ke Delta Lake bersifat atomik. Tidak ada partial write yang merusak data. Mendukung concurrent readers dan writers.

📋 Schema Evolution

Bronze layer menggunakan mergeSchema=true untuk mengakomodasi penambahan kolom baru tanpa break pipeline.

🔄 Upsert (MERGE)

Silver dan Gold menggunakan overwrite per-batch untuk simplifikasi demo. Pada produksi, MERGE INTO direkomendasikan untuk efisiensi.

Cara Menjalankan Pipeline Lakehouse

  1. Bronze — Baca raw JSONL, tambahkan metadata:
    python lakehouse/01_bronze.py
  2. Silver — Cleaning, dedup, schema enforcement:
    python lakehouse/02_silver.py
  3. Gold — Feature store, JOIN, forward-fill:
    python lakehouse/03_gold.py
  4. Export — Konversi ke JSON untuk dashboard:
    python lakehouse/export_to_hdfs.py
  5. Time Travel Demo — Lihat riwayat versi tabel:
    python lakehouse/time_travel_demo.py
